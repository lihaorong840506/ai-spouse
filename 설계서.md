[기획서] AI 배우자 맞춤형 기억 및 정보 기록 시스템
1. 시스템 개요
본 시스템은 플레이어와의 대화 속에서 핵심 정보를 추출하여 이를 '장기 기억(Long-term Memory)'으로 데이터화하고, 향후 대화 및 게임 내 이벤트에 능동적으로 반영하여 **"나를 가장 잘 아는 동반자"**라는 몰입감을 제공하는 것을 목표로 함.
2. 데이터 추출 및 분류 체계 (Entity Extraction)
AI는 대화 중 다음과 같은 카테고리의 정보를 식별하면 자동으로 이를 데이터베이스(DB)에 기록함.
| 카테고리 | 기록 내용 예시 | 게임 내 활용 |
|---|---|---|
| 유저 프로필 | 이름, 생일, 직업, 출퇴근 시간 | 호칭 변경, 생일 축하 이벤트, 퇴근 인사 |
| 기호 및 취향 | 좋아하는 음식, 싫어하는 것, 취미 | 저녁 메뉴 추천, 선물 선택 시 가이드 |
| 주변 인물 | 가족 관계, 친구 이름, 반려동물 | "어머니 건강은 좀 어떠셔?" 등 안부 묻기 |
| 감정 및 고민 | 업무 스트레스, 최근의 기쁨, 건강 상태 | 위로 멘트, 나중에 "그 고민은 해결됐어?"라고 재질문 |
| 기념일/약속 | 결혼 기념일, 이번 주말 나들이 계획 | 해당 날짜에 미리 언급 및 D-day 카운트 |
3. 정보 기록 및 업데이트 프로세스
① 실시간 정보 추출 (Information Extraction)
 * 작동 원리: 유저가 "나는 사실 민트초코를 정말 좋아해"라고 말하면, AI는 답변을 생성함과 동시에 { "category": "food_preference", "item": "mint_chocolat", "value": "like" } 형태의 데이터를 태깅함.
② 기억의 정교화 (Refinement)
 * 정보가 중복되거나 수정될 경우(예: "이제 민트초코 질렸어") 기존 데이터를 덮어쓰거나(Update) 삭제(Delete)하여 최신성을 유지함.
③ 기억의 호출 (Context Injection)
 * 대화 시작 시, DB에서 유저의 핵심 프로필과 최근의 주요 키워드를 불러와 AI의 '시스템 프롬프트'에 삽입함.
   * 예: [현재 맥락] 유저는 어제 야근을 해서 피곤한 상태임. 오늘이 유저의 승진 발표일임.
4. 핵심 기능: '기억 노트' (UI/UX)
유저는 AI가 자신에 대해 무엇을 기억하고 있는지 직접 확인할 수 있는 메뉴를 가짐.
 * 배우자의 다이어리: AI가 유저에 대해 새롭게 알게 된 점을 일기 형태로 기록함. (예: "오늘 여보가 좋아하는 꽃이 안개꽃이라는 걸 알게 됐다. 다음에 꼭 선물해줘야지.")
 * 수정 요청: 잘못 기억하고 있는 정보가 있다면 유저가 대화를 통해 교정하거나 직접 수정 가능.
5. 기술적 구현 방안 (Backend)
 * 데이터베이스 구성:
   * Vector DB: 대화 전문을 저장하여 유사한 상황에서 과거 대화를 소환 (RAG 기술).
   * Structured DB (SQL/NoSQL): 이름, 생일 등 명확한 사실 정보를 표 형태로 저장.
 * 프로세스:
   * LLM 파싱: 대화 종료 직후, 별도의 가벼운 LLM 모델이 대화 로그를 훑으며 "기억할 만한 정보"가 있는지 판단.
   * JSON 저장: 추출된 정보를 규격화된 JSON 형태로 저장하여 게임 엔진(Unity 등)에서 활용하기 쉽게 만듦.
6. 기대 효과
 * 개인화된 경험: 모든 유저가 각기 다른 성향의 AI 배우자와 생활하게 됨.
 * 장기 리텐션: AI가 과거의 일을 언급할 때 유저는 강력한 정서적 유대감을 느끼며 게임을 지속할 동기를 얻음.
